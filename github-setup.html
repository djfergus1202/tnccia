GitHub Repository Setup

Upload your Chemistry Platform to GitHub for collaboration and deployment

1
Create GitHub Repository
Via GitHub Website (Recommended)
  1. Go to GitHub.com and sign in
  2. Click the "+" icon → "New repository"
  3. Repository name: chemistry-education-platform
  4. Description: Advanced AI-powered chemistry platform
  5. Choose Public or Private
  6. Don't add README (we have our own)
  7. License: MIT (recommended)
  8. Click "Create repository"
Via Command Line
# Install GitHub CLI
# Mac: brew install gh
# Linux: sudo apt install gh

# Login and create repo
gh auth login
gh repo create chemistry-education-platform \
  --public \
  --description "Advanced AI-powered chemistry platform"
Copy your repository URL: https://github.com/yourusername/chemistry-education-platform.git
2
Upload Your Files

Choose your preferred upload method:

Method 1: Git Command Line (Recommended)
First time using Git? Configure it first: 
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"
# Navigate to your extracted package
cd path/to/chemistry-platform-github

# Initialize Git repository
git init

# Add all files (everything is ready to upload)
git add .

# Create initial commit
git commit -m "Initial commit: Chemistry Education Platform with JAMA formatting"

# Connect to GitHub repository
git remote add origin https://github.com/yourusername/chemistry-education-platform.git

# Upload to GitHub
git push -u origin main
Method 2: GitHub Desktop (Beginner-Friendly)
  1. Download GitHub Desktop
  2. Sign in with your GitHub account
  3. Click "Clone a repository"
  4. Select your chemistry-education-platform repository
  5. Choose local folder and clone
  6. Copy all files from extracted package to cloned folder
  7. GitHub Desktop detects new files automatically
  8. Add commit message: "Initial commit: Chemistry Platform"
  9. Click "Commit to main"
  10. Click "Push origin"
Method 3: GitHub Web Interface (Limited)
Not recommended - File size limits and doesn't handle folders well
  1. Go to your repository on GitHub
  2. Click "uploading an existing file"
  3. Drag and drop files or click "choose your files"
  4. Add commit message
  5. Click "Commit changes"

4
Configure Repository
Update Repository Information
  1. Go to your repository Settings tab
  2. Update About section:
    • Description: Advanced AI-powered chemistry and pharmacology educational platform
    • Topics: chemistry, education, ai, discord-bot, flask
Enable GitHub Pages (Optional)
  1. Go to Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main
  4. Folder: / (root)
  5. Save
Branch Protection (Optional)
Skip this if you're the only developer. Add later when working with a team.

Security Configuration
Environment Variables Protection

Your package automatically protects sensitive data:

✅ Protected (Never uploaded)
  • .env files
  • node_modules/
  • __pycache__/
  • Log files
  • Generated content
✅ Safe to Upload
  • .env.example (template only)
  • Source code files
  • Documentation
  • Configuration templates
Never commit actual API keys! Always use .env files locally and environment variables in production.
Troubleshooting
Common Issues
# Generate SSH key
ssh-keygen -t rsa -b 4096 -C "your-email@example.com"

# Add to GitHub: Settings → SSH Keys
cat ~/.ssh/id_rsa.pub
# Check file sizes
find . -size +100M

# Remove large files
git rm --cached large-file.zip
git commit -m "Remove large file"
  • Check repository name spelling
  • Verify repository is public or you have access
  • Check remote URL: git remote -v
